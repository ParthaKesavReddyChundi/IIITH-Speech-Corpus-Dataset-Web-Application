# IIITH Speech Corpus Dataset Collection Platform

A production-quality, full-stack web application designed for collecting multilingual (English, Telugu, Hindi) speech recordings. The system is built with a focus on **zero data loss**, ensuring that recordings survive browser crashes, network drops, and session expirations.

## Tech Stack

- **Frontend:** Next.js 16 (App Router), TypeScript, Tailwind CSS, shadcn/ui
- **Backend/Database:** Supabase (PostgreSQL, Auth, Storage, Row-Level Security)
- **Audio Processing:** Client-side via `MediaRecorder` + `ffmpeg.wasm` (WAV conversion)
- **Hosting:** Vercel

## Prerequisites

- Node.js 18+ and npm
- A [Supabase](https://supabase.com/) account (Free tier is sufficient)
- A [Vercel](https://vercel.com/) account (optional, for deployment)

## Local Development Setup

### 1. Supabase Project Setup

1. Create a new project in your Supabase dashboard.
2. Once created, go to **Project Settings -> API** to get your `URL`, `anon` key, and `service_role` key.
3. Copy `.env.example` to `.env.local` and fill in the Supabase variables:
   ```bash
   cp .env.example .env.local
   ```
4. Set up the Database Schema:
   Go to the **SQL Editor** in the Supabase dashboard and run the migrations in order:
   - `supabase/migrations/001_schema.sql`
   - `supabase/migrations/002_rls.sql`
   - `supabase/migrations/003_indexes.sql`
   - `supabase/migrations/004_functions.sql`
   - `supabase/migrations/005_seed.sql`

5. (Optional) Generate TypeScript types:
   If you want to pull the exact types from your live database, run:
   ```bash
   npx supabase login
   npx supabase gen types typescript --project-id YOUR_PROJECT_ID > src/types/database.ts
   ```
   *Note: We have provided a comprehensive manual `src/types/database.ts` to start with.*

### 2. Initial User Setup (Admin & Students)

Since there is no public signup, all users must be created via the Supabase Auth dashboard:

1. Go to **Authentication -> Users** in Supabase and click **Add User -> Create New User**.
2. Create 1 admin user and 7 student users.
3. After creating the users, go to the **Table Editor**, open the `users` table, and insert a row for each user:
   - `auth_uid`: Select the UUID of the auth user you just created.
   - `name`: Enter the user's name.
   - `role`: Select `admin` or `student`.
4. For each **student**, open the `speakers` table and insert a row:
   - `user_id`: Select the ID of the student from the `users` table.

### 3. Storage Setup

1. Go to **Storage** in Supabase and create a new bucket named `recordings`.
2. Do **NOT** make the bucket public. Access will be managed via signed URLs generated by the API.

### 4. Run the Application

Install dependencies and start the development server:

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser to see the application.

## Vercel Deployment

1. Push your code to a GitHub repository.
2. Go to Vercel and import the project.
3. In the Environment Variables section, add all variables from your `.env.local` file.
   - **Important:** Ensure `SUPABASE_SERVICE_ROLE_KEY` is added, as it's required for server-side API routes (e.g., dataset export).
4. Deploy!
